Overview

Urban traffic management relies on lagging indicators and static models. Vehicle-embedded sensors offer a dense, real-time signal that — when processed correctly — predicts congestion 10–20 minutes ahead of occurrence. This system was built in collaboration with Honda R&D Lab and deployed as part of Columbus, Ohio's smart-city initiative.

Data Pipeline

Vehicle Sensors (~4M vehicles, 0.5s intervals) │ ▼ Feature Engineering ├── GIS road network graph ├── Socioeconomic zone factors ├── Historical flow patterns └── Real-time telematics streams │ ▼ Temporal Fusion Transformer + Graph Neural Network │ ▼ Traffic Flow Forecast (5 / 10 / 20 min horizons) │ ▼ Django + Mapbox Real-Time Monitoring Dashboard

Key Results

Achieved 85% accuracy on 10-minute traffic flow prediction horizons. The real-time monitoring dashboard was adopted by the Columbus smart-city operations team. Processing ~4 million vehicle telemetry streams at 0.5-second sampling intervals required careful batching and streaming architecture on GCP Dataflow before feeding the prediction model.
